In **Velo**, syncing is completely automated because the script drives
the performance.\
\
**How Velo Syncs Things Up**

Velo uses your script as a master timeline. Instead of manual \"sync
points,\" the system maps narration directly to browser actions.

- **Script-Driven Movement**

> The **Auto-Flow** agent only moves, clicks, or scrolls when the
> narration reaches the corresponding part of the script.

- **Dynamic Pacing**

> If you add a sentence to a scene, Velo automatically pauses the
> browser agent or extends the visual to match the new audio length.

- **Scene Cards as Anchors**

> Each **Scene Card** acts as a fixed sync block. The \"Visual\"
> instruction and the \"Narration\" text within that card are locked
> together.
